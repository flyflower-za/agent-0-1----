## 🐍 Python 基础

### 为什么需要

整个 Agent 用 Python 写。你需要能读懂示例代码、修改函数、写简单的类。

### 最低够用标准

能读懂并修改下面的代码就算过关：

```python
# 1. 函数：能定义、能调用、能设默认参数
def greet(name: str, greeting: str = "你好") -> str:
    """向用户打招呼"""
    return f"{greeting}，{name}！"

result = greet("张三")
print(result)  # 输出：你好，张三！


# 2. 类：能定义类、能写 __init__、能实例化调用
class ChangeAgent:
    def __init__(self, name: str):
        self.name = name
        self.history = []  # 实例变量，存对话历史

    def ask(self, question: str) -> str:
        self.history.append(question)
        return f"[{self.name}] 正在查询: {question}"


agent = ChangeAgent("变更助手")
print(agent.ask("今天有哪些变更？"))
# 输出：[变更助手] 正在查询: 今天有哪些变更？


# 3. 异步代码：理解 async/await 是干什么的，能照着模板写
import asyncio

async def fetch_data(api: str) -> dict:
    """模拟异步请求 API"""
    await asyncio.sleep(1)  # 假装在等网络响应
    return {"api": api, "status": "ok"}


async def main():
    # 同时发两个请求，不阻塞
    results = await asyncio.gather(
        fetch_data("/change/list"),
        fetch_data("/change/detail")
    )
    print(results)

asyncio.run(main())
```

## 🔧 Git 基本使用

### 为什么需要

你的代码需要用 Git 管理版本。出了 Bug 能回滚，写好了能推送保存。

### 最低够用标准

```bash
# 1. 克隆项目（从远程拉到本地）
git clone https://github.com/yourname/change-agent.git
cd change-agent

# 2. 日常提交三部曲
git status                     # 先看一眼改了啥
git add .                      # 暂存所有改动
git commit -m "feat: 完成 ReAct 循环"   # 提交到本地
git push                       # 推送到远程仓库

# 3. 查看历史
git log --oneline -10          # 看最近 10 条提交

# 4. 出问题了能回退
git checkout -- filename.py    # 放弃单个文件的修改（回到上次提交的状态）
git reset --hard HEAD~1        # 回退到上一个提交（⚠️ 不可恢复，确认再用）
```

### commit message 怎么写

用中文或英文都行，关键是写清楚"做了什么"。建议格式：

```
feat: 完成 ReAct 循环
fix: 修复工具调用死循环问题
docs: 更新前置知识指南
refactor: 把 API 调用抽成独立模块
```

### 速通资源（1-2 小时）

- **可视化交互教程**：https://learngitbranching.js.org/ — 做前 3 关就够
- 注册一个 GitHub 账号，建一个空仓库，把上面的命令走一遍
- `.gitignore` 文件：告诉 Git 哪些文件不要管（`__pycache__/`、`.env`、`node_modules/` 等）

### 本项目用不到，可以暂时不管

分支合并（`git merge`）、rebase、stash、cherry-pick、PR/MR 流程

## 🌐 HTTP/API 概念

### 为什么需要

Agent 核心能力是调用变更系统 API。你需要理解"向某个 URL 发请求 → 拿回 JSON"这件事。

### 核心概念一句话

| 概念 | 一句话理解 |
|------|-----------|
| **REST API** | 用 URL 当"地址"，用 HTTP 方法当"动作"，用 JSON 当"语言" |
| **HTTP 方法** | GET = 查数据，POST = 提交数据，PUT = 改数据，DELETE = 删数据 |
| **JSON** | 一种键值对的数据格式，人和机器都能读 |
| **Endpoint** | 一个 URL 对应一个功能（如 `/api/change/list`） |
| **Token** | 身份凭证，放在请求头里，证明"我是谁" |
| **Status Code** | 200 = 成功，400 = 你请求有问题，401 = 没登录，403 = 没权限，500 = 服务器炸了 |

### 自测：能读懂下面的请求和响应

```python
import requests

# 向变更系统 API 发一个 GET 请求，查今天的变更列表
response = requests.get(
    url="https://change-system.company.com/api/v1/changes",
    headers={
        "Authorization": "Bearer eyJhbGciOiJI...",  # 身份令牌
        "Content-Type": "application/json"
    },
    params={
        "status": "pending",                        # 筛选条件：只看待审批的
        "date_from": "2026-07-07"
    }
)

# 响应返回的 JSON 长这样：
# {
#   "code": 200,
#   "message": "success",
#   "data": {
#     "total": 3,
#     "items": [
#       {"id": "CHG-2026-001", "status": "pending", "title": "数据库升级"},
#       {"id": "CHG-2026-002", "status": "pending", "title": "防火墙规则更新"},
#       {"id": "CHG-2026-003", "status": "pending", "title": "应用配置变更"}
#     ]
#   }
# }

# 把 JSON 转成 Python 字典，然后操作它
data = response.json()              # JSON → Python dict
print(data["code"])                 # 200
print(data["data"]["total"])        # 3

changes = data["data"]["items"]     # 取列表
for change in changes:              # 遍历每个变更
    print(f"{change['id']}: {change['title']}")

# 输出：
# CHG-2026-001: 数据库升级
# CHG-2026-002: 防火墙规则更新
# CHG-2026-003: 应用配置变更
```

### 速通资源（2-3 小时）

- 仔细看一遍上面的代码，理解每一行在干什么
- 打开浏览器访问 https://jsonplaceholder.typicode.com/todos/1 — 这是一个免费假 API，返回 JSON
- 用 `requests` 库（先 `pip install requests`）自己写一个脚本，调通至少一个公开 API：
  ```python
  import requests
  resp = requests.get("https://jsonplaceholder.typicode.com/todos/1")
  print(resp.json())
  # 输出：{'userId': 1, 'id': 1, 'title': 'delectus aut autem', 'completed': False}
  ```
- 如果公司有变更系统的 API 文档，翻一下，看有哪些接口、返回什么字段、怎么认证

### 本项目用不到，可以暂时不管

GraphQL、gRPC、WebSocket、OAuth 2.0 的四种授权模式、JWT 原理
