## 📋 前置知识清单

开始之前，确保你对以下内容有基本了解（不要求精通）：

- [ ] **Python 基础** — 能读写函数、类、异步代码
- [ ] **命令行操作** — 能在终端跑脚本、装依赖
- [ ] **Git 基本使用** — clone、commit、push
- [ ] **HTTP/API 概念** — 知道什么是 REST API、JSON

## 第 1 周：理解 LLM 与 Agent 的本质

**目标：搞清楚 LLM 是什么、Agent 是什么、两者什么关系**

### 理论
- [ ] LLM 的基本原理：Token、Context Window、Temperature
- [ ] Prompt Engineering 基础：System Prompt、Few-shot、Chain-of-Thought
- [ ] 什么是 AI Agent？与 Chatbot 的本质区别
- [ ] Agent 的核心架构：感知 → 推理 → 行动 → 记忆（Perception → Reasoning → Action → Memory）
- [ ] ReAct 模式（Reasoning + Acting）

### 实践
- [ ] 用 OpenAI / 智谱 API 跑通第一个 "Hello World" 对话
- [ ] 手写一个 System Prompt，让模型扮演特定角色
- [ ] 体验现有 Agent 产品：OpenClaw、Cursor、Claude Code，感受 Agent 的能力边界

### 📝 本周产出
- 笔记：`01-llm-and-agent-fundamentals.md`
- 代码：`hello-llm.py` — 调通至少一个 LLM API

## 第 2 周：Function Calling — 让 LLM 能"动手"

**目标：理解并实现 Function Calling（工具调用），这是 Agent 的核心能力**

### 理论
- [ ] Function Calling 的工作机制：模型如何决定调用哪个函数
- [ ] Tool / Function 的定义规范（JSON Schema）
- [ ] 单工具调用 vs 并行多工具调用
- [ ] 工具调用的上下文管理（结果如何回传给模型）

### 实践
- [ ] 手写 3 个工具函数（如：查天气、算数学、读文件）
- [ ] 用 OpenAI / 智谱 API 实现 Function Calling 完整循环
- [ ] 让 Agent 自主选择调用哪个工具
- [ ] 处理工具调用失败的情况（错误兜底）

### 📝 本周产出
- 笔记：`02-function-calling.md`
- 代码：一个能自主调用 3+ 工具的命令行 Agent

## 第 3 周：RAG — 给 Agent 装上"知识库"

**目标：掌握检索增强生成（RAG），让 Agent 能基于私有知识回答问题**

### 理论
- [ ] 为什么需要 RAG？LLM 的知识截止问题
- [ ] Embedding 与向量数据库的概念
- [ ] 文档分块策略（Chunking）：固定大小、语义分块
- [ ] 相似度检索：余弦相似度、Top-K
- [ ] RAG 的完整流程：加载 → 分块 → 嵌入 → 存储 → 检索 → 生成

### 实践
- [ ] 用 LangChain 或 LlamaIndex 搭建一个最简 RAG
- [ ] 向自己的文档库提问（可以用 Obsidian 笔记做数据源）
- [ ] 对比不同 chunk size 对回答质量的影响
- [ ] 引入重排序（Reranker）提升检索质量（可选）

### 📝 本周产出
- 笔记：`03-rag-basics.md`
- 代码：一个能对自己笔记/文档进行问答的 RAG 程序

## 第 4 周：记忆与状态管理

**目标：让 Agent 拥有短期记忆和长期记忆，能持续跟踪上下文**

### 理论
- [ ] Agent 记忆的分类：短期（对话上下文）vs 长期（持久化存储）
- [ ] Context Window 的管理策略：滑动窗口、摘要压缩
- [ ] 对话历史存储方案：内存、文件、数据库
- [ ] Vector Memory：用向量检索实现"回忆"
- [ ] Entity Memory：记住人物、事件、关系

### 实践
- [ ] 实现一个有记忆的对话 Agent，能记住 10 轮以上的上下文
- [ ] 当对话超出 Context Window 时，自动摘要压缩
- [ ] 用 SQLite 或 JSON 文件持久化对话历史
- [ ] 实现一个简单的 Vector Memory（基于 Embedding + 相似度检索）

### 📝 本周产出
- 笔记：`04-memory-and-state.md`
- 代码：一个有长期记忆的 Agent，关掉重开还能"记住"你

- [ ] 加入人工确认环节（Agent 执行关键操作前先问人）

## 第 7 周：实战项目 — 从设计到实现

**目标：综合前 6 周所学，从零设计并实现一个完整的 Agent 项目**

### 项目选择（选一个）

**选项 A：个人知识助手**
- 读取 Obsidian 笔记
- 支持语义搜索 + 智能问答
- 能自动生成笔记摘要和关联推荐
- 有 Web 界面

**选项 B：代码 Review Agent**
- 读取 Git 仓库
- 自动分析代码质量、安全漏洞
- 生成 Review 报告
- 能提交 GitHub PR 评论

**选项 C：自动化工作流 Agent**
- 监听特定事件（如邮件、消息）
- 自动执行多步骤任务
- 生成报告并推送通知
- 支持人工审核介入

- [ ] 逐模块实现
- [ ] 联调测试
